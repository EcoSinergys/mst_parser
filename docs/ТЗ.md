# Техническое Задание (ТЗ)

## Парсер каталога насосов MST Pumps

**Версия:** 2.0  
**Дата:** 16 июня 2026  
**Статус:** Реализовано

---

## 1. Назначение программы

Программа предназначена для автоматического сбора (парсинга) каталога насосного оборудования с сайта **mstpumps.com** с последующим экспортом данных в форматы, пригодные для импорта в CMS **MODX 3**.

---

## 2. Термины и определения

| Термин | Определение |
|--------|-------------|
| **Парсинг** | Автоматический сбор данных с веб-страниц |
| **Product List** | Страница со списком товаров (paginated list) |
| **Категория** | Раздел каталога (например, "Slurry Pumps") |
| **Подкатегория** | Подраздел категории (например, "Heavy Duty Slurry Pump") |
| **SKU** | Артикул товара |
| **MODX 3** | CMS, в которую импортируются данные |
| **gzip/brotli** | Алгоритмы сжатия HTTP-ответов |

## 3. Функциональные требования

### 3.1. Режимы работы

#### Режим A — Верификация через productlist
- Сбор **всех** ссылок на продукты со страниц productlist (1-81)
- На каждой странице ожидается **12 продуктов** (кроме последней — 3)
- **Ожидаемое количество:** 963 продукта
- Используется для полной верификации каталога

#### Режим B — Парсинг по дереву категорий
- Обход каталога по структуре: Категория → Подкатегория → Продукты
- Предопределённые категории (см. п. 4)
- Автоматическое определение страниц пагинации

### 3.2. Собираемые данные

Для каждого продукта собираются:
- **Название** (Title)
- **URL** страницы продукта
- **Описание** (HTML-контент)
- **Характеристики** (ключ-значение)
- **Изображения** (small + large)
- **Product ID** (из URL)
- **Alias** (для MODX)

### 3.3. Форматы вывода

1. **catalog_structured.json** — иерархическая структура:
   ```json
   {
     "categories": [{
       "category_name": "Slurry Pumps",
       "category_url": "https://...",
       "subcategories": [{
         "subcategory_name": "Heavy Duty Slurry Pump",
         "subcategory_url": "https://...",
         "products": [...]
       }]
     }]
   }
   ```

2. **modx_import.json** — плоский список для MODX 3:
   ```json
   [{
     "pagetitle": "High Head Multi Impeller Pump",
     "alias": "high-head-multi-impeller-pump",
     "content": "<p>...</p>",
     "template": 1,
     "published": true,
     "product_image": "downloaded_images/small/...jpg",
     "product_category": "Multistage Water Pump",
     "source_url": "https://..."
   }]
   ```

### 3.4. Загрузка изображений
- Изображения скачиваются в папку `downloaded_images/`
- Структура: `small/` и `large/`
- Имена файлов: `{product_id}_{index}.jpg`
- Пропускается при флаге `--skip-images`

## 5. Защита от блокировок (Anti-Ban)

### 5.1. Ротация User-Agent
- Пул из **10 современных User-Agent** (Chrome, Firefox, Safari, Edge)
- Случайный выбор для каждого запроса

### 5.2. Задержки между запросами
- **Min delay:** 1.5 секунды
- **Max delay:** 4 секунды
- Случайное значение в диапазоне

### 5.3. Retry-механизм
- При ошибках сети: **до 5 попыток**
- Exponential backoff: 2, 4, 8... секунд + jitter
- При статусах 429/503: дополнительная пауза 5-15 секунд
- При 0 товаров на странице: **до 3 повторных попыток**

### 5.4. HTTP-заголовки
- `Accept-Language: ru-RU,ru;q=0.9,...`
- `Accept: text/html,...`
- `Sec-Fetch-*` заголовки (как браузер)
- `Referer` обновляется после каждого перехода

## 6. Технические требования

### 6.1. Язык и окружение
- **Язык:** Go 1.25+
- **Сборка:** `go build -o mst_parser.exe .`
- **ОС:** Windows 10/11

### 6.2. Зависимости
| Пакет | Версия | Назначение |
|-------|--------|------------|
| `github.com/PuerkitoBio/goquery` | v1.12.0 | Парсинг HTML |
| `golang.org/x/net` | v0.52.0 | Сетевые возможности |

### 6.3. Параметры командной строки

| Флаг | Тип | Значение по умолч. | Описание |
|------|-----|-------------------|----------|
| `--mode` | string | `B` | Режим: `A` (productlist) или `B` (категории) |
| `--category` | string | `""` | URL конкретной категории |
| `--limit` | int | `0` | Лимит продуктов (0 = без лимита) |
| `--skip-images` | bool | `false` | Пропустить скачивание изображений |

### 6.4. Обработка ошибок
- Корректная обработка 404 (страница не найдена)
- Пропуск недоступных продуктов с предупреждением
- Продолжение работы при ошибках в отдельных элементах
- Логирование всех ошибок в stdout

## 7. Архитектура программы

### 7.1. Модули (файлы)

| Файл | Ответственность |
|------|-----------------|
| **main.go** | Точка входа, CLI-флаги, оркестрация |

```
       ├─ catalog.go (HTML → ссылки)
       ├─ product.go (HTML → Product)

       └─ storage.go (Product → JSON)
```

## 9. Ограничения и известные проблемы

1. **Проблема сжатия:** Сервер mstpumps.com может использовать brotli-сжатие, которое Go не всегда корректно обрабатывает. Решение: multiple retry с разными заголовками Accept-Encoding.
2. **Динамическая пагинация:** Некоторые категории могут использовать JavaScript-пагинацию, что требует иного подхода к сбору ссылок.
3. **Смена структуры сайта:** При изменении HTML-шаблонов сайта парсер может потребовать обновления CSS-селекторов.
4. **Лимит времени:** Полный парсинг всех 963 продуктов занимает несколько часов из-за обязательных задержек.
